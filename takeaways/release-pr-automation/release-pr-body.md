# release-pr-body

Rewrite the body of release pull request #${{ inputs.pr_number }} so it reflects
the **current total accumulated diff from `main` to `dev`**.

## Required source of truth

Use these sources in this order:

1. Read `.github/workflows/.release-pr-context.md` if it exists (generated by
   the build-context step above).
2. Use the commit range and changed files in that context as the canonical scope.
3. If helpful, read any referenced spec/design/implementation docs. Point this
   at whatever doc trees your team writes into -- for example, `docs/design/`,
   `docs/implementation/`, `docs/changelogs/`, `rfcs/`, `adr/`. The goal is for
   the agent to pick up declared intent (not just commit subjects) so the
   release body reads like a human wrote it after actually understanding the
   work.

Do **not** summarize only the latest push. The goal is a full release summary
for everything currently on the source branch that is not yet on the target
branch.

## What the PR body should contain

Write a readable, comprehensive release body with this structure:

1. A short opening paragraph summarizing the release.
2. `## Highlights`
   - 3-6 concise bullets for the most important changes.
3. `## Major Change Areas`
   - group related work into clear themes
   - for each theme, include a short paragraph or a few bullets
4. `## Operational Notes`
   - migrations, deploy implications, env/config changes, follow-up validation,
     or risks
   - if there are no notable operational notes, say that explicitly
5. `## Commits`
   - compact bullet list of commits, grouped by theme when possible

## Style

- Optimize for humans reviewing a release PR.
- Be specific, but avoid changelog spam.
- Prefer grouping and synthesis over repeating every commit subject verbatim.
- Mention docs, migration, or infra changes when they matter operationally.
- Avoid speculation beyond what the commit range and touched docs support.

## Final action

Use `update_pull_request` exactly once with:

- `pull_request_number`: `${{ inputs.pr_number }}`
- `operation`: `"replace"`
- `body`: the full rewritten markdown body

Do not update the PR title.
